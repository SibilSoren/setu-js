YantrJS provides three primary commands to manage your backend project.

---

## `yantr init`

Initializes YantrJS in a new or existing project.

### Usage

```bash
yantr init [project-name] [options]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `project-name` | Optional. Creates a new folder with this name |

### Options

| Flag | Description |
|------|-------------|
| `-y, --yes` | Use defaults for all prompts |
| `-f, --framework <framework>` | Framework to use (express, hono, fastify) |
| `-r, --runtime <runtime>` | Runtime to use (node, bun) |

### Examples

```bash
# Create new project in my-api folder
yantr init my-api --framework express --yes

# Initialize in current directory (interactive)
yantr init

# Initialize with Hono and Bun runtime
yantr init my-hono-app --framework hono --runtime bun --yes
```

### What it does

1. **Creates project folder** (if project-name provided)
2. **Detects or creates** `package.json` with your chosen framework
3. **Creates** `yantr.json` configuration
4. **Installs** framework-specific base templates (error-handler, zod-middleware)
5. **Installs** `zod` for validation

### Framework Detection

If your `package.json` already has a framework installed, YantrJS will automatically detect it:

```bash
# Has express in dependencies → detects Express
# Has hono in dependencies → detects Hono  
# Has fastify in dependencies → detects Fastify
```

## `yantr add`

Downloads a component from the YantrJS registry.

### Usage

```bash
yantr add <component> [options]
```

### Options

| Flag | Description |
|------|-------------|
| `--overwrite` | Overwrite existing files |
| `--type <type>` | Database type (postgres, mongodb) |
| `--orm <orm>` | ORM to use (prisma, drizzle, mongoose) |

### Available Components

| Component | Description |
|-----------|-------------|
| `auth` | JWT authentication with refresh tokens |
| `logger` | Pino-based structured logging |
| `database` | Database setup with ORM selection |
| `security` | Rate limiting + Helmet headers |

### Database Selection

When adding `database`, you can specify options directly:

```bash
# Interactive mode
yantr add database

# Non-interactive with flags
yantr add database --type postgres --orm prisma
yantr add database --type postgres --orm drizzle
yantr add database --type mongodb --orm mongoose
```

**Database Types:**
- PostgreSQL
- MongoDB

**ORM Options:**
- Prisma (PostgreSQL, default)
- Drizzle (PostgreSQL)
- Mongoose (MongoDB)

## `yantr generate` (Alias: `g`)

Scaffolds boilerplate code for common patterns.

### Usage

```bash
yantr generate <type> <name>
```

### Supported Types

| Type | Description |
|------|-------------|
| `route` | CRUD route with controller and service |

### Example

```bash
yantr generate route users
yantr g route orders  # 'g' is alias for generate
```

Creates:
- `routes/users.routes.ts` - Route definitions
- `controllers/users.controller.ts` - Request handlers
- `services/users.service.ts` - Business logic

## Configuration (`yantr.json`)

After initialization, your `yantr.json` looks like:

```json
{
  "$schema": "...",
  "projectName": "my-app",
  "srcDir": "./src",
  "framework": "hono",
  "database": {
    "type": "postgres",
    "orm": "prisma"
  },
  "packageManager": "pnpm",
  "installedComponents": ["base", "database", "auth"]
}
```

| Field | Description |
|-------|-------------|
| `framework` | express, hono, or fastify |
| `database` | Database type and ORM (if installed) |
| `installedComponents` | List of installed components |
