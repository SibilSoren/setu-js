YantrJS provides four primary commands to manage your backend project.

---

## `yantr create`

Creates a new project with a full interactive wizard.

### Usage

```bash
yantr create <project-name> [options]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `project-name` | Required. Name of the project folder to create |

### Options

| Flag | Description |
|------|-------------|
| `-y, --yes` | Use defaults for all prompts |
| `-f, --framework <framework>` | Framework: express, hono, fastify |
| `-r, --runtime <runtime>` | Runtime: node, bun |
| `-t, --db-type <type>` | Database type: postgres, mongodb, none |
| `--orm <orm>` | ORM: prisma, drizzle, mongoose |

### Interactive Wizard

When run without `--yes`, the wizard guides you through:

1. **Runtime** — Node.js or Bun
2. **Framework** — Express, Hono, or Fastify
3. **Database** — PostgreSQL, MongoDB, or skip
4. **ORM** — Prisma, Drizzle, or Mongoose (based on database)
5. **Components** — Auth, Logger, Security (multi-select)
6. **Package Manager** — npm, pnpm, yarn, or bun

### Examples

```bash
# Full interactive wizard
yantr create my-api

# Quick start with defaults (Express + Node.js)
yantr create my-api --yes

# Create with specific stack
yantr create my-api --framework hono --runtime bun --db-type postgres --orm prisma

# Create with Express and skip database
yantr create my-api --framework express --db-type none --yes
```

### What it does

1. **Creates project folder** with your chosen name
2. **Creates** `package.json` with framework and runtime settings
3. **Creates** `yantr.json` configuration
4. **Installs** base templates (error-handler, zod-middleware)
5. **Optionally sets up** database with your chosen ORM
6. **Optionally adds** selected components (auth, logger, security)
7. **Installs** all dependencies at once

## `yantr init`

Initializes YantrJS in an **existing** Node.js project.

### Usage

```bash
yantr init [options]
```

### Options

| Flag | Description |
|------|-------------|
| `-y, --yes` | Use defaults for all prompts |
| `-f, --framework <framework>` | Framework: express, hono, fastify |

### Examples

```bash
# Initialize in current project (interactive)
yantr init

# Initialize with defaults
yantr init --yes

# Initialize with specific framework
yantr init --framework hono
```

### What it does

1. **Detects** your framework from `package.json` (or asks)
2. **Creates** `yantr.json` configuration
3. **Installs** framework-specific base templates
4. **Installs** `zod` for validation

### Framework Detection

If your `package.json` already has a framework installed, YantrJS will automatically detect it:

```bash
# Has express in dependencies → detects Express
# Has hono in dependencies → detects Hono  
# Has fastify in dependencies → detects Fastify
```

> [!NOTE]
> If no `package.json` exists, `yantr init` will prompt you to use `yantr create` instead.

## `yantr add`

Downloads a component from the YantrJS registry.

### Usage

```bash
yantr add <component> [options]
```

### Options

| Flag | Description |
|------|-------------|
| `--overwrite` | Overwrite existing files |
| `--type <type>` | Database type (postgres, mongodb) |
| `--orm <orm>` | ORM to use (prisma, drizzle, mongoose) |

### Available Components

| Component | Description |
|-----------|-------------|
| `auth` | JWT authentication with refresh tokens |
| `logger` | Pino-based structured logging |
| `database` | Database setup with ORM selection |
| `security` | Rate limiting + Helmet headers |

### Database Selection

When adding `database`, you can specify options directly:

```bash
# Interactive mode
yantr add database

# Non-interactive with flags
yantr add database --type postgres --orm prisma
yantr add database --type postgres --orm drizzle
yantr add database --type mongodb --orm mongoose
```

**Database Types:**
- PostgreSQL
- MongoDB

**ORM Options:**
- Prisma (PostgreSQL, default)
- Drizzle (PostgreSQL)
- Mongoose (MongoDB)

## `yantr generate` (Alias: `g`)

Scaffolds boilerplate code for common patterns.

### Usage

```bash
yantr generate <type> <name>
```

### Supported Types

| Type | Description |
|------|-------------|
| `route` | CRUD route with controller and service |

### Example

```bash
yantr generate route users
yantr g route orders  # 'g' is alias for generate
```

Creates:
- `routes/users.routes.ts` - Route definitions
- `controllers/users.controller.ts` - Request handlers
- `services/users.service.ts` - Business logic

## Configuration (`yantr.json`)

After initialization, your `yantr.json` looks like:

```json
{
  "$schema": "...",
  "projectName": "my-app",
  "srcDir": "./src",
  "framework": "hono",
  "database": {
    "type": "postgres",
    "orm": "prisma"
  },
  "packageManager": "pnpm",
  "installedComponents": ["base", "database", "auth"]
}
```

| Field | Description |
|-------|-------------|
| `framework` | express, hono, or fastify |
| `database` | Database type and ORM (if installed) |
| `installedComponents` | List of installed components |
